# Звонки в чатах (Audio & Video)

## ✅ ЧТО ДОБАВЛЕНО:

### 1. **WebRTC соединение**
- Peer-to-peer аудио и видео связь
- ICE серверы Google STUN

### 2. **Сокет сигналинг**
- События для инициации звонка
- Обмен SDP offer/answer
- Обмен ICE candidates

### 3. **UI звонков**
- Кнопки 📞 (аудио) и 📹 (видео) в header чата
- Экран входящего звонка
- Экран активного звонка
- Управление (мут, камера, завершить)

### 4. **Только для персональных чатов**
- Групповые звонки пока не поддерживаются

## 📁 СОЗДАННЫЕ ФАЙЛЫ:

### 1. **`mobile/src/components/CallScreen.js`**
Компонент экрана звонка:

**Props:**
- `visible` - видимость модального окна
- `isIncoming` - входящий звонок?
- `isVideo` - видео звонок?
- `callerName` - имя звонящего
- `onAccept` - принять звонок
- `onDecline` - отклонить звонок
- `onEnd` - завершить звонок
- `localStream` - локальный медиа поток
- `remoteStream` - удаленный медиа поток

**Функции:**
- Отображение видео (локальное + удаленное)
- Таймер звонка
- Управление микрофоном (мут/анмут)
- Управление камерой (вкл/выкл)

### 2. **`mobile/src/hooks/useWebRTC.js`**
Хук для WebRTC логики:

**Возвращает:**
```javascript
{
  localStream,      // MediaStream локального пользователя
  remoteStream,     // MediaStream удаленного пользователя
  isCallActive,     // Активен ли звонок
  incomingCall,     // Данные входящего звонка
  makeCall,         // Функция инициации звонка
  answerCall,       // Функция ответа на звонок
  endCall,          // Функция завершения звонка
  declineCall       // Функция отклонения звонка
}
```

**Основные функции:**

1. **makeCall(chatId, recipientId, isVideo)**
   - Запрашивает доступ к камере/микрофону
   - Создает RTCPeerConnection
   - Генерирует SDP offer
   - Отправляет через сокет

2. **answerCall(callData)**
   - Запрашивает доступ к камере/микрофону
   - Создает RTCPeerConnection
   - Генерирует SDP answer
   - Отправляет через сокет

3. **endCall()**
   - Останавливает все треки
   - Закрывает peer connection
   - Уведомляет удаленную сторону

### 3. **`backend/src/socket/chatSocket.js`**
Добавлены обработчики звонков:

**События:**

- `ice_candidate` - обмен ICE candidates
- `call_end` - завершение звонка
- `call_decline` - отклонение звонка

### 4. **`mobile/src/screens/ChatScreen.js`**
Интеграция звонков:

**Добавлено:**
- useWebRTC хук
- Кнопки 📞 и 📹 в header
- CallScreen компонент
- Определение recipientId

## 🎯 КАК РАБОТАЕТ:

### Инициация звонка:
```
1. Пользователь A нажимает 📞 или 📹
2. makeCall(chatId, recipientId, isVideo)
3. Запрашивается доступ к камере/микрофону
4. Создается RTCPeerConnection
5. Генерируется SDP offer
6. Socket: call_initiate → Backend → Пользователь B
7. У пользователя B появляется входящий звонок
```

### Ответ на звонок:
```
1. Пользователь B нажимает "Принять"
2. answerCall(callData)
3. Запрашивается доступ к камере/микрофону
4. Создается RTCPeerConnection
5. Устанавливается удаленный SDP offer
6. Генерируется SDP answer
7. Socket: call_answer → Backend → Пользователь A
8. Начинается обмен ICE candidates
9. Устанавливается P2P соединение
10. Медиа потоки начинают передаваться
```

### WebRTC Flow:
```
A: createOffer() → setLocalDescription(offer)
   ↓ socket
B: setRemoteDescription(offer) → createAnswer() → setLocalDescription(answer)
   ↓ socket
A: setRemoteDescription(answer)
   
Обмен ICE:
A/B: onicecandidate → socket → addIceCandidate
```

## ⚠️ ВАЖНО:

### Требования:
1. **HTTPS** для production (WebRTC требует)
2. **Разрешения браузера** на камеру/микрофон
3. **Только Web** (React Native mobile не поддерживается пока)
4. **Только персональные чаты** (не групповые)

### Ограничения:
- WebRTC работает только между двумя пользователями
- Для групповых звонков нужен Media Server (Janus, Jitsi)
- Мобильные платформы требуют react-native-webrtc

### Firewall:
Если соединение не устанавливается:
- Проверьте firewall
- Используйте TURN сервер (не только STUN)
- Добавьте в ICE_SERVERS:
```javascript
{
  urls: 'turn:your-turn-server.com:3478',
  username: 'user',
  credential: 'password'
}
```
