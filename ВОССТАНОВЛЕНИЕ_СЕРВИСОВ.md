## 1️⃣ RAILWAY.APP - Backend + Database

### Что проверить:

#### A. Найти проект
```
1. Зайдите: https://railway.app/dashboard
2. Найдите проект (возможные названия):
   - polka
   - chat-app
   - messenger
   - polka-production
```

#### B. Проверить сервисы в проекте

Должно быть 2 сервиса:

**1. PostgreSQL Database**
- Тип: Database
- Статус: должен быть Active ✅
- Важно: Скопируйте DATABASE_URL из вкладки "Connect"

**2. Backend Service**
- Тип: GitHub Repo / Empty Service
- Статус: должен быть Active ✅
- Root Directory: `/backend` (проверьте в Settings)

#### C. Проверить переменные окружения Backend

Settings → Variables → должны быть:

```env
DATABASE_URL = postgresql://postgres:xxx@postgres.railway.internal:5432/railway
JWT_SECRET = your-random-secret-key-12345
NODE_ENV = production
FRONTEND_URL = https://polka-pi.vercel.app
PORT = 3000
```

⚠️ **Если какой-то переменной нет - добавьте!**

#### D. Получить URL Backend

```
Settings → Networking → Generate Domain (если нет)
Скопируйте URL, например:
https://polka-production.up.railway.app
```

#### E. Проверить работает ли

Откройте в браузере:
```
https://polka-production.up.railway.app
```

Должен вернуть:
```json
{"message": "Polka Messenger API Server", "version": "1.0.0"}
```

✅ **Если работает - отлично!**
❌ **Если не работает:**
- Проверьте логи: Deployments → View Logs
- Проверьте DATABASE_URL правильный
- Попробуйте Redeploy

#### F. Если проект не найден

**Создайте новый:**
```
1. New Project → Deploy from GitHub
2. Подключите ваш GitHub (нужен репозиторий)
3. Add Database → PostgreSQL
4. Add Service → выберите репозиторий
5. Settings → Root Directory: /backend
6. Variables → добавьте все переменные (см. выше)
7. Deploy
```

## 2️⃣ VERCEL.COM - Frontend

### Что проверить:

#### A. Найти проект
```
1. Зайдите: https://vercel.com/dashboard
2. Найдите проект (возможные названия):
   - polka
   - polka-pi
   - chat-app
   - messenger
```

#### B. Проверить настройки

Settings → General:
```
Framework Preset: Other (или Create React App)
Root Directory: mobile
Build Command: npx expo export:web
Output Directory: web-build
Node.js Version: 18.x (рекомендуется)
```

#### C. Проверить переменные окружения

Settings → Environment Variables:

```env
REACT_APP_API_URL = https://polka-production.up.railway.app/api
REACT_APP_BACKEND_URL = https://polka-production.up.railway.app
NODE_ENV = production
```

⚠️ **Замените URL на ваш Railway URL!**

#### D. Получить URL Frontend

```
Deployments → Production → Visit
Или на главной странице проекта
Например: https://polka-pi.vercel.app
```

#### E. Проверить работает ли

Откройте в браузере:
```
https://polka-pi.vercel.app
```

Должно открыться приложение с формой входа/регистрации.

✅ **Если работает - отлично!**
❌ **Если не работает:**
- Проверьте логи: Deployments → Function Logs
- Проверьте REACT_APP_BACKEND_URL правильный
- Попробуйте Redeploy

#### F. Обновить CORS на Railway

Важно! Вернитесь в Railway → Backend → Variables:
```
FRONTEND_URL = https://polka-pi.vercel.app (ваш Vercel URL)
```

**Redeploy Backend!**

#### G. Если проект не найден

**Создайте новый:**
```
1. Add New → Project
2. Import Git Repository (нужен GitHub)
3. Configure:
   - Framework: Other
   - Root Directory: mobile
   - Build Command: npx expo export:web
   - Output Directory: web-build
4. Environment Variables: добавьте все (см. выше)
5. Deploy
```

## 3️⃣ CLOUDINARY - Файлы (Опционально)

### Что проверить:

#### A. Войти в аккаунт
```
1. Зайдите: https://cloudinary.com/users/login
2. Войдите или создайте аккаунт (бесплатно)
```

#### B. Получить API ключи

Dashboard → Account Details (справа сверху):
```
Cloud Name:  dxxxxx (скопируйте)
API Key:     123456789012345 (скопируйте)
API Secret:  xxxxxxxxxxxxx (нажмите "Show" → скопируйте)
```

#### C. Добавить в Railway

Railway → Backend → Variables → New Variable:
```
CLOUDINARY_CLOUD_NAME = ваш cloud name
CLOUDINARY_API_KEY = ваш api key
CLOUDINARY_API_SECRET = ваш api secret
```

**Redeploy Backend!**

#### D. Добавить локально

Backend → `.env` файл:
```env
CLOUDINARY_CLOUD_NAME=ваш_cloud_name
CLOUDINARY_API_KEY=ваш_api_key
CLOUDINARY_API_SECRET=ваш_api_secret
```

**Перезапустите Backend!**

## 4️⃣ GITHUB (если нужно задеплоить)

### Если у вас еще нет репозитория:

#### A. Создать репозиторий
```
1. Зайдите: https://github.com/new
2. Repository name: polka-messenger
3. Private или Public (на выбор)
4. Create repository
```

#### B. Загрузить код
```bash
cd C:\Users\Gamer\chat-app

# Инициализировать git
git init

# Добавить .gitignore (уже есть)
# Добавить файлы
git add .

# Первый коммит
git commit -m "Initial commit - Polka messenger"

# Подключить репозиторий
git remote add origin https://github.com/ВАШ_USERNAME/polka-messenger.git

# Загрузить
git branch -M main
git push -u origin main
```

#### C. Подключить к Railway и Vercel

После загрузки на GitHub:
- Railway → Import from GitHub (выберите репозиторий)
- Vercel → Import from GitHub (выберите репозиторий)
