# Новые функции Polka Messenger

## Реализованные функции

### 1. ✅ Удаление чатов

**Два варианта удаления:**
- **Удалить у себя** - чат скрывается только у вас (is_hidden = true)
- **Удалить для всех** - чат удаляется полностью у обоих пользователей (только для приватных чатов)

**API:**
```
DELETE /api/chats/:id
Body: { "forEveryone": true/false }
```

### 2. ✅ Запрет дубликатов чатов

Теперь невозможно создать два приватных чата между одними и теми же пользователями. При попытке создать дубликат:
- Возвращается существующий чат
- Если чат был скрыт у пользователя, он автоматически становится видимым

### 3. ✅ Фото для групповых чатов

Добавлено поле `avatar_url` в модель Chat. Владелец группы может устанавливать/менять аватар.

**API:**
```
PUT /api/chats/:id
Body: { "avatar_url": "URL" }
```

### 4. ✅ Управление группой только для владельца

Добавлено поле `owner_id` в Chat. Только владелец может:
- Удалять участников группы
- Изменять название и аватар группы

**API:**
```
DELETE /api/chats/:id/members/:userId
```

### 5. ✅ Выход из группы

Любой участник может покинуть группу. Если выходит владелец, права автоматически передаются следующему участнику.

**API:**
```
POST /api/chats/:id/leave
```

### 6. ✅ Временные отступы и разделители дат

**В чате теперь отображаются:**
- Разделители дат ("Сегодня", "Вчера", "25 марта 2026")
- Увеличенные отступы между сообщениями, если прошло более 10 минут

## Необходимые шаги

### 1. Выполнить миграцию БД

Выполните SQL миграцию на вашем сервере (Railway или локально):

```bash
# Подключитесь к PostgreSQL
psql -U your_username -d your_database

# Выполните миграцию
\i backend/migrations/add_owner_and_hidden.sql
```

Или вручную:
```sql
-- Добавляем owner_id в чаты
ALTER TABLE chats ADD COLUMN IF NOT EXISTS owner_id UUID REFERENCES users(id);

-- Добавляем is_hidden в chat_members
ALTER TABLE chat_members ADD COLUMN IF NOT EXISTS is_hidden BOOLEAN DEFAULT false;

-- Обновляем существующие групповые чаты
UPDATE chats 
SET owner_id = (
  SELECT user_id 
  FROM chat_members 
  WHERE chat_members.chat_id = chats.id 
    AND chat_members.role = 'admin' 
  ORDER BY chat_members.joined_at ASC 
  LIMIT 1
)
WHERE type = 'group' AND owner_id IS NULL;
```

### 2. Обновить фронтенд API

Нужно добавить функции в `mobile/src/services/api.js`:

```javascript
export const chatAPI = {
  // ... существующие функции
  
  leaveGroup: (chatId) => api.post(`/chats/${chatId}/leave`),
  
  hideChatForMe: (chatId) => api.delete(`/chats/${chatId}`, { 
    data: { forEveryone: false } 
  }),
  
  deleteChatForEveryone: (chatId) => api.delete(`/chats/${chatId}`, { 
    data: { forEveryone: true } 
  })
};
```

### 3. Обновить UI

**Добавить в экран информации о чате (ChatInfo):**
- Кнопку "Удалить чат у себя"
- Кнопку "Удалить чат для всех" (только для приватных чатов)
- Кнопку "Покинуть группу" (для групповых чатов)
- Возможность загрузки аватара группы (только для владельца)
- Список участников с кнопкой удаления (только для владельца)

**Добавить в экран управления группой (GroupManage):**
- Отображение владельца группы
- Кнопки удаления участников (только для владельца)

## Тестирование

1. Создайте приватный чат с пользователем
2. Попробуйте создать второй чат с тем же пользователем - должен открыться существующий
3. Удалите чат "у себя" - он должен скрыться только у вас
4. Создайте группу - вы должны стать владельцем
5. Попробуйте удалить участника - должно работать только у владельца
6. Добавьте несколько сообщений с интервалом >10 минут - должны появиться отступы
7. Отправьте сообщения в разные дни - должны появиться разделители дат

## Примечание

После деплоя на Railway и Vercel все изменения вступят в силу автоматически. Не забудьте выполнить миграцию БД!
