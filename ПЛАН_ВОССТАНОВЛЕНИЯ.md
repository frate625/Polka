## 📋 ПОШАГОВАЯ ИНСТРУКЦИЯ

### ШАГ 1: Запустить локальную базу данных PostgreSQL

#### 1.1 Запустить PostgreSQL службу

**Вариант А: Через Services (Службы)**
```
1. Win + R → введите: services.msc
2. Найдите: "postgresql-x64-XX" (где XX - версия)
3. Правый клик → Start (Запустить)
```

**Вариант Б: Через pgAdmin4**
```
1. Откройте pgAdmin4
2. Подключитесь к серверу (localhost)
3. Если не подключается - служба не запущена
```

#### 1.2 Создать базу данных через pgAdmin4

```
1. Откройте pgAdmin4
2. Подключитесь к PostgreSQL (localhost)
3. Правый клик на "Databases" → Create → Database
4. Name: chat_app
5. Owner: postgres
6. Save
```

**Или через SQL:**
```sql
CREATE DATABASE chat_app;
```

#### 1.3 Проверить подключение

**Параметры подключения:**
```
Host: localhost
Port: 5432
Database: chat_app
Username: postgres
Password: [ваш пароль при установке]
```

### ШАГ 2: Настроить локальный Backend

#### 2.1 Обновить .env файл

Откройте: `C:\Users\Gamer\chat-app\backend\.env`

Убедитесь что там:
```env
# Database
DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/chat_app

# Server
PORT=3000
NODE_ENV=development

# JWT Secret
JWT_SECRET=super-secret-jwt-key-for-development-12345

# CORS - Frontend URL
FRONTEND_URL=http://localhost:19006

# Redis (опционально - можно закомментировать)
# REDIS_URL=redis://localhost:6379
```

⚠️ **ВАЖНО:** Замените `ВАШ_ПАРОЛЬ` на пароль PostgreSQL!

#### 2.2 Установить зависимости и запустить

```bash
cd C:\Users\Gamer\chat-app\backend
npm install
node src/config/initDatabase.js
npm start
```

**Должно появиться:**
```
✅ Database connection established successfully.
✅ Database models synchronized.
🚀 Server running on http://localhost:3000
```

### ШАГ 5: Проверить Railway.app

#### 5.1 Проверить статус проекта

1. Зайдите на: https://railway.app/
2. Войдите в свой аккаунт
3. Найдите проект "polka" или "chat-app"

#### 5.2 Проверить что там есть:

**Должны быть 2 сервиса:**
- 🗄️ **PostgreSQL** - база данных
- 🚀 **Backend** - Node.js сервер

#### 5.3 Проверить переменные окружения

В Backend сервисе → Variables:
```
DATABASE_URL = (должен быть автоматически из PostgreSQL)
JWT_SECRET = your-random-secret-key-12345
NODE_ENV = production
FRONTEND_URL = https://polka-pi.vercel.app (или ваш URL)
```

#### 5.4 Проверить URL backend

Settings → Networking → Public URL:
```
Должен быть: https://polka-production.up.railway.app
или похожий
```

#### 5.5 Проверить работает ли

Откройте в браузере:
```
https://polka-production.up.railway.app
```

Должен вернуть JSON:
```json
{"message": "Polka Messenger API Server", "version": "1.0.0"}
```

## 🚨 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: PostgreSQL не запускается

**Решение:**
```
1. Откройте pgAdmin4
2. Если просит пароль - это Master Password для pgAdmin
3. После входа подключитесь к серверу PostgreSQL
4. Если ошибка подключения - проверьте службу в services.msc
```

### Проблема 2: Backend не подключается к БД

**Решение:**
```
1. Проверьте пароль в .env файле
2. Проверьте что БД chat_app создана в pgAdmin4
3. Проверьте что PostgreSQL запущен
```

### Проблема 3: Frontend не подключается к Backend

**Решение:**
```
1. Проверьте что Backend запущен (http://localhost:3000)
2. Проверьте REACT_APP_BACKEND_URL в mobile/.env
3. Перезапустите expo: Ctrl+C → npx expo start --web --clear
```

### Проблема 4: Railway проект не найден

**Решение:**
```
1. Возможно проект был удален
2. Создайте новый проект на Railway
3. Следуйте инструкции из DEPLOYMENT_GUIDE.md
```

### Проблема 5: Vercel проект не найден

**Решение:**
```
1. Возможно проект был удален
2. Создайте новый проект на Vercel
3. Следуйте инструкции из DEPLOYMENT_GUIDE.md
```
